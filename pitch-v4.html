01 · The Big Bet

我们押的不是 AI 工具，是组织形态本身的迁移

旧公司是围绕「人作为执行者」构建的——招人、培训、汇报、KPI、996。

未来公司是围绕「Agent 作为执行者」构建的——配置、调度、监督、迭代、7×24。

不用 Agent 的公司，不是 Agent 不行，是公司终将被淘汰。

02 · The Shift

过去 200 年的组织设计，全部是为「人的局限性」服务的

  • 人会累 → 需要排班和轮岗
  • 人有信息瓶颈 → 需要会议和同步
  • 人有专业边界 → 需要部门和职能墙
  • 人有招聘周期 → 需要 HC 和 headcount 预算
  • 人会跳槽 → 需要知识管理和交接

Agent 没有以上任何一个限制。所以围绕 Agent 构建的组织，结构会和今天截然不同——更扁平、更动态、更弹性、更接近「愿景 → 产出」的直线。

Anthropic 开始裁员的那一天，可能就是因为他们装了 Soloco。

05 · Pillar I · 自动驾驶样板

为什么先讲自动驾驶

  • 工具链高度 SOP 化：bazel / argo workflow / replay / cli / DBC 自动生成——每支团队都在和确定性管道打交道
  • 产出可批量验证：数值对比、回归测试、覆盖率指标——AI 产出可机器校验，不靠人审美
  • 团队规模大 + 烧钱压力大：100-500 人工程团队，估值压力下最有动力 AI 化
  • 真实可触达：我们就在 tranxmart 平台里工作，wiki 24 个领域 ~190 篇笔记是现成 SOP 弹药

07 · Pillar III · 自迭代飞轮

组织一旦形成，越用越值

  • 跑通的任务 → 沉淀为客户专属 SOP
  • 出错的环节 → Conductor 自动调整任务单 / 引入新 Agent 角色
  • 老板的拍板偏好 → Orchestrator 自动学习，未来同类决策不再打扰
  • 跨客户最佳实践 → 平台抽象成可订阅的「团队模板」

真正的护城河不是初始功能，是组织记忆的不可迁移性。
