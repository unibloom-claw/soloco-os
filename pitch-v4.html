01TL;DR — The Big Bet

Soloco 不是一个 AI 工具，是 AI 组织的操作系统。

旧公司是围绕「人作为执行者」构建的——招人、培训、汇报、KPI、年会、996。未来公司是围绕「Agent 作为执行者」构建的——配置、调度、监督、迭代、运行 7×24。

我们押的不是某个 AI 产品，是组织形态本身的迁移。

不用 Agent 的公司，不是 Agent 不行，是公司终将被淘汰。

02大胆论断 · 新组织形态已迫在眉睫

过去 200 年的公司组织设计——部门、汇报线、OKR、组织架构图——全部是为「人这种执行者」的局限性服务的：

  • 人会累 → 需要排班和轮岗
  • 人有信息瓶颈 → 需要会议和同步
  • 人有专业边界 → 需要部门和职能墙
  • 人有招聘周期 → 需要 HC 和 headcount 预算
  • 人会跳槽 → 需要知识管理和交接

Agent 没有以上任何一个限制。所以围绕 Agent 构建的组织，结构会和今天的公司截然不同——更扁平、更动态、更可弹性伸缩、更接近「愿景 → 产出」的直线。

谁先转型谁活，否则被淘汰

这不是「采用 AI 工具」级别的变化，是组织 DNA 级别的变化。

上一次同等量级的迁移：手工业作坊 → 工业化公司。蒸汽机不是工具，是组织重构的扳机。拒绝转型的公司不会「少了点效率」，是整个赛道被新组织形态从底层吃掉。

03暴论 · Anthropic 也是我们的客户 😏

Anthropic 现在在急速扩张——招人、扩 team、铺产品线。但他们终将面对一个尴尬事实：

Reflexive Product Validation

做 foundation model 的公司，自己内部却用人手做大量可被 AI 化的工作——研究方向调研、安全评估报告、客户成功、文档维护、行政审批……

Anthropic 开始裁员的那一天，可能就是因为他们装了 Soloco。

这不是嘲讽——这是 reflexive 产品验证：如果连 Anthropic 都用我们，那 AI 组织的迁移就是真的；如果他们不用，说明这个故事还没成熟。我们押的是前者。

05支柱一 · 业务接管（以自动驾驶公司为样板）

为什么先讲自动驾驶

  • 工具链已经高度 SOP 化：bazel / argo workflow / replay / cli / DBC 自动生成 / param_identification 跑批——每一支团队都在和确定性管道打交道
  • 产出可批量验证：数值对比、回归测试、覆盖率指标——AI 产出可以用机器校验，不靠人审美
  • 团队规模大、烧钱压力大：一家自动驾驶公司 100-500 人工程团队，估值压力下最有动力 AI 化
  • 真实可触达：我们自己就在 tranxmart 平台里工作，wiki 里 24 个领域、~190 篇笔记是现成的 SOP 弹药

07支柱三 · 自迭代飞轮（一旦形成，越用越值）

普通 SaaS 是装好就停在那里的。AI 组织是装好后会自己长大的。

  • 跑通的任务 → 沉淀为该公司专属 SOP
  • 出错的环节 → Conductor 自动调整任务单 / 引入新 Agent 角色
  • 老板的拍板偏好 → Orchestrator 自动学习，未来同类决策不再打扰
  • 跨客户的最佳实践 → 平台抽象成可订阅的「团队模板」

结果：客户用了 6 个月的 Soloco 组织 ≠ 客户用了 1 个月的 Soloco 组织。切换成本随时间指数级上升。

真正的护城河不是初始功能，是组织记忆的不可迁移性。
类比：你不会因为有更好的 IDE 就把整个 GitHub 仓库迁走——同样的逻辑放大 100 倍。

12投资人必问

Q1. 这听起来太大了，先做什么？

自动驾驶 Repo + Build/CI Takeover 一个切入口。用户群最清晰、技术债最熟、演示张力最强。6 个月内 10 个付费客户做 A 轮叙事。大愿景说给投资人听，小切口做给客户看。

Q2. 为什么 Anthropic / OpenAI 不会做？

  • 战略错配：他们是 foundation model 公司，毛利结构决定他们做模型 + 通用工具，不做 50+ 行业 SOP
  • 用户错配：他们的产品基因是 builder-first，不是企业管理者
  • 跨模型立场：Soloco 跑在 Claude 和 GPT 之上，Anthropic 永远不会主动让客户跨模型
  • 更重要：他们也会变成我们的客户（见 §03） 😏

Q3. 为什么不是 Devin / Manus 升级一下就做了？

他们的产品基因是「单 Agent 完成任务」。变成「组织接管职能」要重写：调度内核（单→多 Agent）、产品形态（chat→组织画布+甘特图）、GTM（开发者自助→行业垂直销售）。升级成本 ≈ 重做一个产品。

Q4. 老板真的会用一张组织画布管公司吗？

现在大公司老板已经在用 Tableau / Looker 看运营数据。Soloco 组织画布 = 运营数据 + 可执行 Agent——看数据时直接拍板让对应 Agent 调整。这是数据 dashboard 的自然进化。

Q5. ROI 怎么算？

单包定价 = 替代团队成本的 10-20%。AV 公司 200 人工程团队 ¥6000 万-1 亿/年 → Soloco 跨包组合卖 ¥1000-2000 万/年替代 30-50% SOP 工作——ROI 立竿见影 + 弹性扩缩 + 7×24 不休。

Q6. 信任和敏感数据问题？

  • 本地 NPM 部署，Agent 跑在客户内网
  • 用户自带 API key，模型供应商不接触客户业务数据
  • Orchestrator GO/NO-GO Gate 把关键决策回到人手上
  • 全过程审计日志 + Pivot 可追溯

Q7. 如果客户用了三个月发现 AI 干不了怎么办？

我们用 Pivot Protocol——出错不重试，自动换 fallback target 或换 task order。客户看到的是「AI 主动求帮助」，不是「AI 静默失败」。信任来自治理可见性，不是吹效果。

Q8. 为什么是你们？

  • Cofounder 技术背书 + 主创全职推进
  • 已有 20 万行代码、跑通 Orchestrator 编排模式、takeover demo 可现场演示
  • 5 月底 NPM 公开发布——不是 PPT 公司

14一段话总结（背熟版）

我们押的不是 AI 工具，是组织形态本身的迁移——围绕人执行者构建的旧公司，正在被围绕 Agent 执行者构建的新公司替代。Soloco 做三件事：

（1）业务接管——把每一支手头工作密集的职能团队 AI 化，从自动驾驶公司的 Repo / 标定 / 仿真团队开始，扩展到任何「软件工程为核心」的行业；

（2）Orchestrator 单一入口——老板提供数据和愿景，Soloco 用组织画布和甘特图把执行状态全部可视化；

（3）自迭代飞轮——组织一旦形成会持续自迭代，客户用得越久切换成本越高。

商业模式：垂直接管包年订阅，单价 = 替代团队成本的 10-20%，本地部署 + 用户自带 key 决定我们零 token 成本、毛利结构天然好。终局：每家公司的组织都跑在 Soloco 上——包括 Anthropic 自己。
